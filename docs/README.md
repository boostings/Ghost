# Ghost Documentation

This folder supplements `PLAN.md` with the lightweight project artifacts that make the software engineering process visible in the repository.

- `requirements/software-requirements.md`: scoped functional and quality requirements.
- `architecture/adr-001-shared-whiteboards-and-verified-answers.md`: architectural decision record for the core domain rules.
- `testing/test-strategy.md`: test pyramid, commands, and coverage gates.
- `runbooks/local-development.md`: developer setup and verification runbook.

`PLAN.md` remains the implementation source of truth. These documents exist to show how the plan is translated into requirements, architecture, testing, and operational guidance.
